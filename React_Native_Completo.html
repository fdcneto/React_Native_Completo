2. A Revolução da Bridge (Ponte)

Antes do React Native: Android (Java/Kotlin) e iOS (Swift/Objective-C) eram mundos separados. Duas equipes, código duplicado, custos dobrados.

React Native: Escreva em JavaScript, renderize componentes nativos de verdade via ponte.

+-------------------+ +-------------------+ +-------------------+ | JAVASCRIPT | | BRIDGE | | NATIVE SIDE | | (React Code) |----->| (JSON/JSI) |----->| (iOS/Android) | +-------------------+ +-------------------+ +-------------------+ | | | v v v +-------------------+ +-------------------+ +-------------------+ | Hermes/JSC | | Serialização | | Native Modules | | JS Engine | | Mensagens JSON | | UI Components | +-------------------+ +-------------------+ +-------------------+

Como Funciona a Bridge

1. JavaScript Thread

Seu código React roda aqui. Componentes, hooks, lógica de negócio.

2. Bridge (Ponte)

Canal assíncrono que serializa mensagens JSON entre JS e nativo.

3. Native Thread

Executa código nativo real: renderiza views, acessa APIs do sistema.

⚠️
Implicação importante: Operações pesadas no JavaScript (loops grandes, filtros complexos) podem travar a UI pois compartilham a mesma thread. Use Web Workers ou mova para nativo quando necessário.

React Native vs Híbrido vs Nativo

Caracteristica Nativo Híbrido (Ionic) React Native
Performance Excelente Limitada Boa a Excelente
Componentes UI Nativos WebView Nativos
Base de Codigo Separada Unica Unica
Curva de Aprendizado Alta Baixa Media

3. Os 3 Pilares do React Native

📦

Componentes

Funções que retornam interface. Ex: <Botão />, <Card />. Letra MAIÚSCULA obrigatória.

⬇️

Props

Parâmetros que o pai envia para o filho. Imutáveis (quem recebe só lê).

🧠

State

"Memória" do componente. Quando muda, a tela re-renderiza automaticamente.

Componentes Essenciais do React Native

Componente Analogia Web Função Principal
<View> <div> Container para layout (obrigatório ter um pai)
<Text> <p>, <span> Exibir texto (obrigatório para qualquer texto)
<TextInput> <input> Campo de entrada de texto
<TouchableOpacity> <button> Área clicável com feedback visual
<Image> <img> Exibir imagens (locais ou remotas)
<ScrollView> overflow: scroll Container com rolagem
<FlatList> - Lista performática com virtualização

Seu Primeiro Componente

// App.js - Estrutura básica de um componente import React from 'react'; import { View, Text, StyleSheet } from 'react-native'; export default function App() { return ( <View style={styles.container}> <Text style={styles.title}> Olá, React Native! </Text> </View> ); } const styles = StyleSheet.create({ container: { flex: 1, justifyContent: 'center', alignItens: 'center', backgroundColor: '#f5f5f5', }, title: { fontSize: 28, fontWeight: 'bold', color: '#1a2639', }, });
Regra de Ouro: Todo componente React Native deve ter um único elemento pai. Use <View> para envolver múltiplos elementos.

Árvore de Componentes

Um aplicativo React é uma árvore de componentes onde tudo começa no App - o componente principal. Todos os outros componentes estão dentro dele, direta ou indiretamente.

App (Raiz) ├── Header ├── Lista │ ├── Item 1 │ ├── Item 2 │ └── Item 3 └── Botão

Relação Pai e Filho:

  • App é pai de Header, Lista e Botão
  • Lista é pai de Item
  • Item é filho de Lista

Em React, essa hierarquia é importante porque:

  • O pai pode enviar dados para o filho (via props)
  • O filho não alterá o pai diretamente

Regras do JSX

3 Regras Fundamentais:
  1. Tudo precisa estar dentro de um elemento pai
  2. Código JS vai entre chaves { }
  3. Componentes começam com letra MAIÚSCULA

4. Estado: A Memória do Componente

O state é onde guardamos dados que mudam ao longo do tempo. Quando o state muda, o React re-renderiza automaticamente o componente com os novos valores.

useState: O Hook Fundamental

import React, { useState } from 'react'; import { View, Text, Button, StyleSheet } from 'react-native'; export default function Contador() { // useState retorna [valorAtual, funçãoParaAtualizar] const [count, setCount] = useState(0); const incrementar = () => { setCount(count + 1); }; return ( <View style={styles.container}> <Text style={styles.count}>{count}</Text> <Button title="+1" onPress={incrementar} /> </View> ); }

State vs Props: Quando Usar Cada Um?

Caracteristica Props State
Origem Vem do componente pai Interno do componente
Mutabilidade Imutavel (so leitura) Mutavel (via setState)
Controle Quem passa controla Componente controla
Quando usar Configuração, dados iniciais Interácao do usuário
⚠️
Nunca faça isso: contador = 5 ou contador++
Sempre use: setContador(5) ou setContador(prev => prev + 1)
TL;DR - Props vs State:

Props configuram. State reage.

5. Formulários e Entrada de Dados

O TextInput é o componente de entrada de texto. No React Native, usamos o padrão componente controlado: o valor do input vem do state, e ao digitar atualizamos o state.

Componente Controlado

const [nome, setNome] = useState(''); <TextInput value={nome} // valor vem do state onChangeText={setNome} // ao digitar, atualiza state placeholder="Digite seu nome" style={styles.input} />

Props Importantes do TextInput

Prop Função
keyboardType Tipo de teclado: "email-address", "numeric", "phone-pad"
secureTextEntry Oculta texto (para senhas)
autoCapitalize "none", "words", "characters"
multiline Permite múltiplas linhas
maxLength Limite de caracteres

Componente Controlado Explicado

Quando o valor do TextInput vem do state é o state e atualizado pelo TextInput, chamamos isso de Componente Controlado.

Tipo Quem guarda o valor?
Input comum O campo guarda o valor
Input controlado O valor vive no state. O campo apenas exibe.

Eventos em React Native

Evento é uma função que executa quando algo acontece. React e orientado a eventos - você não manda atualizar, você reage ao que acontece.

Evento Quando dispara
onChangeText Quando o usuário digita
onPress Quando o usuário toca/clica

TextInput vs HTML input

HTML input (Web)

  • Elemento do HTML
    • Só existe no navegador
  • JSX vira HTML

TextInput (React Native)

  • Componente nativo
  • JSX -> Bridge -> Nativo
  • Android: EditText
  • iOS: UITextField

Erros Comuns com Formulários

⚠️
  • Esquecer de usar value no TextInput
  • Tentar alterár o state direto (use sempre setState)
  • Esquecer que state é assíncrono
  • Misturar muita lógica dentro do JSX
TL;DR - Formulários:

React não é sobre telas. React é sobre Estado → Interface é consequência. Se o estado estiver correto, a interface se ajusta sozinha.

6. Flexbox: Organizando a Interface

No React Native, toda View já tem display: flex por padrão. Os elementos filhos obedecem automaticamente às regras do container.

Propriedades Principais

Propriedade Função Valores Comuns
flexDirection Eixo principal "column" (vertical), "row" (horizontal)
justifyContent Alinhamento no eixo principal "flex-start", "center", "space-between"
alignItens Alinhamento no eixo secundario "stretch", "center", "flex-start"
flex Proporção de espaço Número (1, 2, 3...)

Exemplos Práticos

Centralizar Perfeitamente +
container: { flex: 1, justifyContent: 'center', // vertical (se column) alignItens: 'center', // horizontal }
Duas Colunas Iguais +
<View style={{ flexDirection: 'row' }}> <View style={{ flex: 1, backgroundColor: 'lightblue' }} /> <View style={{ flex: 1, backgroundColor: 'lightgreen' }} /> </View>

As 3 Perguntas do Flexbox

Flexbox responde a tres perguntas fundamentais:

  1. Em qual direcao os elementos ficam? → flexDirection
  2. Como eles se distribuem no eixo principal? → justifyContent
  3. Como eles se alinham no eixo secundario? → alignItens

Resumo: flexDirection e Eixos

flexDirection justifyContent (principal) alignItens (secundario)
column (padrão) Vertical Horizontal
row Horizontal Vertical

flex: 1 - Ocupando Espaco

O valor flex: 1 faz o componente ocupar todo o espaço disponível.

const styles = StyleSheet.create({ container: { flex: 1, // Ocupa tela inteira backgroundColor: '#f5f5f5', }, header: { flex: 1, // 1 parte do espaço backgroundColor: 'blue', }, content: { flex: 2, // 2 partes (dobro) backgroundColor: 'white', }, });

7. Navegação Entre Telas

Apps reais possuem múltiplas telas. A biblioteca React Navigation é o padrão da indústria para navegar entre telas no React Native.

Instalação

npm install @react-navigation/native @react-navigation/native-stack npx expo install react-native-screens react-native-safe-área-context

Estrutura Básica

import { NavigationContainer } from '@react-navigation/native'; import { createNativeStackNavigator } from '@react-navigation/native-stack'; const Stack = createNativeStackNavigator(); function HomeScreen({ navigation }) { return ( <View> <Text>Tela Home</Text> <Button title="Ir para Perfil" onPress={() => navigation.navigate('Perfil')} /> </View> ); } export default function App() { return ( <NavigationContainer> <Stack.Navigator initialRouteName="Home"> <Stack.Screen name="Home" component={HomeScreen} /> <Stack.Screen name="Perfil" component={PerfilScreen} /> </Stack.Navigator> </NavigationContainer> ); }

Métodos de Navegação

Metodo Função
navigate('Tela', params) Vai para tela (empilha)
goBack() Volta para tela anterior
replace('Tela') Substitui tela atual (não empilha)
reset({ index: 0, routes }) Reseta a pilha de navegacao

Passando Dados Entre Telas

Para enviar dados de uma tela para outra, use o segundo parametro de navigate:

// Enviando dados navigation.navigate('Home', { nome: 'Maria', idade: 25 }); // Recebendo dados na outra tela function HomeScreen({ route }) { const { nome, idade } = route.params; return ( <View> <Text>Ola, {nome}!</Text> </View> ); }

8. Hooks Avançados

Alem do useState, React oferece vários hooks para diferentes cenarios. Dominar esses hooks e essencial para construir apps profissionais.

useEffect: Efeitos Colateráis

import { useEffect, useState } from 'react'; function UserProfile({ userId }) { const [user, setUser] = useState(null); useEffect(() => { // Executa quando userId muda fetchUser(userId).then(setUser); // Função de limpeza (opcional) return () => console.log('Limpeza'); }, [userId]); // Array de dependencias }

Array de Dependencias

Configuração Quando Executa
useEffect(() => {}) Apos TODA renderizacao
useEffect(() => {}, []) Apenas na montagem
useEffect(() => {}, [dep]) Na montagem e quando 'dep' mudar

useMemo e useCallback: Otimizacao

import { useMemo, useCallback } from 'react'; // Memoriza resultado de calculo pesado const sortedItens = useMemo(() => { return itens.sort((a, b) => a.name.localeCompare(b.name)); }, [itens]); // Memoriza função (evita re-render dos filhos) const handleItemClick = useCallback((id) => { onItemClick(id); }, [onItemClick]);

10. Nova Arquitetura do React Native

A Nova Arquitetura representa o maior salto tecnológico do React Native desde sua criação. Ela substitui a Bridge tradicional por sistemas mais eficientes: JSI, Fabric e TurboModules.

A Bridge Tradicional e Suas Limitacoes

❌ Bridge Antiga

  • Serializacao JSON obrigatoria
  • Comunicacao apenas assincrona
  • Fila unica (gargalo)
  • Todos módulos carregados no inicio

✅ Nova Arquitetura

  • Acesso direto via JSI
  • Chamadas sincronas e assincronas
  • Comunicacao paralela
  • Lazy loading de módulos

JSI - JavaScript Interface

O JSI permite que JavaScript acesse diretamente objetos C++ sem serializacao.

JSI - JavaScript Interface
+-------------------+ +-------------------+ | JAVASCRIPT | | NATIVO (C++) | | (Hermes/JSC) | | | +-------------------+ +-------------------+ | | | +---------------------------+ | +--->| JSI RUNTIME |<------+ | +---------------------+ | | | Host Functions | | <-- Chamadas diretas | | Host Objects | | <-- Zero-copy | | Memory Shared | | <-- Sincrono | +---------------------+ | +---------------------------+

Fabric - Novo Sistema de Renderizacao

1. Render Phase (Background)
↓
2. Calcula nova Shadow Tree
↓
3. Commit Phase (Atomic)
↓
4. Mount Phase (Native)
💡
Beneficio prático: Com Fabric, atualizações de alta prioridade (input do usuário) podem interromper renders de baixa prioridade, melhorando a responsividade.

11. Otimizacao de Performance

Performance e critica em apps mobile. Usuários esperam 60 FPS fluidos, resposta instantanea a toques e carregamento rápido.

Ferramentas de Profiling

Flipper

Inspector completo para React Native: layout, rede, banco de dados, logs e performance.

React DevTools

Visualize a árvore de componentes e identifique re-renders desnecessários.

Hermes Profiler

Perfil de CPU e memória do engine JavaScript Hermes.

Problemas Comuns e Solucoes

1. Re-renders Desnecessários +

Solucao: Use React.memo, useMemo e useCallback:

// Componente memoizado const UserCard = React.memo(({ user, onPress }) => { return ( <TouchableOpacity onPress={onPress}> <Text>{user.name}</Text> </TouchableOpacity> ); }); // Hook para memoizar valores const sortedItens = useMemo(() => { return itens.sort((a, b) => a.name.localeCompare(b.name)); }, [itens]);
2. Listas Lentas +

Solucao: Use FlatList com virtualizacao:

<FlatList data={itens} keyExtractor={(item) => item.id} renderItem={renderItem} removeClippedSubviews={true} maxToRenderPerBatch={10} windowSize={21} initialNumToRender={10} getItemLayout={(data, index) => ({ length: ITEM_HEIGHT, offset: ITEM_HEIGHT * index, index, })} />

ScrollView vs FlatList: Quando Usar?

Caracteristica ScrollView FlatList
Complexidade Simples Mais avançado
Carregamento Carrega tudo na memória Carrega so o visivel
Ideal para Poucos itens Listas grandes
Performance Pode ficar lento Mais performatico
⚠️
Atenção: O ScrollView carrega todos os elementos na memória ao mesmo tempo. Se tivermos 1000 ou 5000 itens, o aplicativo pode ficar lento. Por isso, use FlatList para listas grandes!

Checklist de Performance

Área Verificar Ferramenta
JavaScript Re-renders desnecessários, memory leaks React DevTools, Flipper
Lista Virtualizacao, keyExtractor, getItemLayout Flipper Layout
Imagens Tamanho, cache, formato Flipper Network
Native Bridge overhead, view hierarquia Instruments, Android Profiler

12. Animações Avancadas

Animações fluidas sao essenciais para uma experiência mobile premium. O React Native oferece Reanimated 3 para animacoes de alta performance.

Reanimated 3: Animações na UI Thread

npm install react-native-reanimated

Animação Básica com Shared Values

import Animated, { useSharedValue, useAnimatedStyle, withSpring } from 'react-native-reanimated'; function AnimatedCard() { const scale = useSharedValue(1); const animatedStyle = useAnimatedStyle(() => { return { transform: [{ scale: scale.value }], }; }); const handlePressIn = () => { scale.value = withSpring(0.95); }; const handlePressOut = () => { scale.value = withSpring(1); }; return ( <Pressable onPressIn={handlePressIn} onPressOut={handlePressOut}> <Animated.View style={[styles.card, animatedStyle]}> <Text>Card Animado</Text> </Animated.View> </Pressable> ); }
💡
Performance tip: Reanimated 3 roda na UI Thread, então as animacoes continuam fluidas mesmo se a JS Thread estiver ocupada.

Contador com Limites

Fácil

Objetivo: Criar um contador que:

  • Inicia em 0
  • Tem botoes +1 e -1
  • Não permite valores negativos (min: 0)
  • Não permite valores acima de 100 (max: 100)
  • Exibe mensagem quando atinge os limites
Ver Solucao +
import React, { useState } from 'react'; import { View, Text, TouchableOpacity, StyleSheet, Alert } from 'react-native'; export default function ContadorLimitado() { const [count, setCount] = useState(0); const [data, setData] = useState(null); const [loading, setLoading] = useState(true); const [error, setError] = useState(null); const [isFromCache, setIsFromCache] = useState(false); const requestIdRef = useRef(0);
